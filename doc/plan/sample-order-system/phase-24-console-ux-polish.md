# Phase 24: 콘솔 UI/UX 개선

계층: view

## 배경

Phase 22/21의 `view/console_view.py`는 최소 골격이라 메뉴/출력이 밋밋하다 (단순 텍스트 나열, 목록 정렬 없음, 구분선 없음). 로직(controller/model)은 건드리지 않고 콘솔 출력 형식만 다듬는다. Phase21에서 주문/모니터링/출고/생산라인 관련 함수도 추가됐으므로 이번 Phase는 그것들도 포함한다.

## 목표

`view/console_view.py`의 출력 형식을 개선한다 — 로직/데이터 흐름은 그대로 두고, 사람이 보기 편하게 정렬/구분선/표 형태 등을 적용한다. 비즈니스 로직은 절대 추가하지 않는다 (여전히 입출력만).

## 설계

- `show_main_menu()`: 메뉴 항목을 구분선으로 감싸고, 번호 정렬을 맞춘다.
- `show_sample()` / `show_samples()`: 시료 목록을 표 형태(고정폭 컬럼: 시료ID/이름/평균생산시간/수율)로 정렬해 출력한다. 목록이 여러 건일 때 헤더 행을 추가한다.
- `show_order()`: 주문 정보를 구분선 있는 표 형태로 정리 (주문ID/고객명/시료ID/수량/상태).
- `show_status_counts()`: 상태별 카운트를 표 형태로.
- `show_stock_judgement()`: 재고 판정 결과를 표 형태로, 상태값에 따라 접두어(여유/부족/고갈)를 명확히 구분.
- `show_production_lines()`: 생산중/대기중 섹션을 구분선으로 나누고, 진행률을 `n/m (p%)` 형태로 보여준다.
- `show_message()`: 안내/에러 메시지에 `[안내]`/`[오류]` 같은 접두어를 붙여 일반 목록 출력과 구분되게 한다.
- 함수 시그니처(입력 파라미터, 반환값)는 바꾸지 않는다 — `controller/main_controller.py`가 호출하는 방식은 그대로 유지해 다른 계층에 영향 없게 한다.

## 추가 개선: rich 라이브러리 도입

1차 개선(고정폭 문자열 정렬)이 여전히 밋밋하다는 피드백에 따라, 외부 라이브러리 `rich`를 도입해 실제 터미널 색상/박스/표를 사용한다.

- 의존성: `requirements.txt`에 `rich` 추가 (이 프로젝트 첫 외부 의존성 — 기존 "표준 라이브러리만 사용" 원칙에서 벗어남, README에 설치 안내 필요).
- `view/console_view.py`를 `rich.console.Console` + `rich.table.Table` + `rich.panel.Panel` 기반으로 재작성:
  - 메인 메뉴: `Panel`로 감싸고 제목에 색상.
  - 시료/주문/모니터링/생산라인 목록: `rich.table.Table`로 실제 테두리+헤더 색상.
  - 안내(`[안내]`)는 청록색, 오류(`[오류]`)는 빨간색으로 스타일링.
  - 재고판정(여유/부족/고갈)은 상태별 색상(여유=초록, 부족=노랑, 고갈=빨강).
- 함수 시그니처는 그대로 유지 (controller는 영향 없음).

- [x] 기존 테스트 전체 회귀 확인
