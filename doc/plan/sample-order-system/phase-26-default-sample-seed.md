# Phase 26: 기본 제공 시료/재고 초기 데이터

계층: main

## 배경

지금까지는 시료를 하나도 등록 안 한 빈 상태로 시작해야 해서, 실행해보려면 매번 직접 등록부터 해야 했다. 처음 실행해도 바로 살펴보고 시험해볼 수 있도록 기본 시료 몇 종을 미리 등록해둔다.

## 목표

`main.py` 최초 실행 시 (아직 시료가 하나도 등록 안 된 상태라면) 기본 시료 5종과 각각의 초기 재고를 자동으로 등록한다. 이미 등록된 시료가 있으면(=한 번이라도 실행한 적 있으면) 건드리지 않는다.

## 설계

- `main.py`에 `seed_default_samples(sample_repo, inventory_repo)` 함수 추가: `sample_repo.find_all()`이 비어있을 때만 아래 기본 시료 5종 + 재고를 등록한다.
- 기본 시료 목록 (시료ID, 이름, 평균생산시간, 수율, 초기재고):
  - S-001, 실리콘 웨이퍼-8인치, 0.5, 0.92, 480
  - S-002, GaN 에피택셜-4인치, 0.3, 0.78, 220
  - S-003, SiC 파워기판-6인치, 0.8, 0.92, 30
  - S-004, 포토레지스트-PR7, 0.2, 0.95, 910
  - S-005, 산화막 웨이퍼-SiO2, 0.6, 0.88, 0
- `SampleRepository`에 `find_all(self) -> list[Sample]`가 없다면 추가 필요 (현재 `find_by_id`/`search`만 있음) — "시료가 하나도 없는지" 판단하려면 전체 목록 조회가 필요하다.

## 완료 조건

- [ ] `SampleRepository.find_all()` 테스트
- [ ] 시료가 비어있을 때 `seed_default_samples` 호출 시 5종이 등록되는지 테스트
- [ ] 이미 시료가 하나라도 있으면 `seed_default_samples`가 아무것도 하지 않는지 테스트
- [ ] `python main.py`로 최초 실행 시 시료관리 메뉴에서 5종이 바로 조회되는지 수동 확인
