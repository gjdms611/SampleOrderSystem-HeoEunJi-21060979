# Phase 35: 승인/거절/취소 진입 시 대상 주문 목록 먼저 표시 + 서브메뉴 취소 옵션

## 배경

1. 주문 승인(2)/거절(3)/취소(4)를 선택하면 바로 "주문ID: " 프롬프트가 떠서, 지금 뭘 승인/거절/취소할 수 있는지 보지 못한 채 order_id를 외워서 입력해야 한다. Phase33의 "승인대기 조회"는 별도 메뉴로만 접근 가능해 여전히 불편하다.
2. 시료관리/주문/모니터링의 서브메뉴("1. 등록 2. 조회 3. 검색" 등)에는 명시적인 "취소" 옵션이 없다 — 빈 입력이나 "0"을 넣어도 그냥 "잘못된 선택입니다"만 뜬다(결과적으로 메인 메뉴로는 돌아가지만 취소라는 의도가 안내되지 않는다).

## 목표

- 승인/거절/취소 액션 선택 시, order_id를 묻기 전에 현재 RESERVED 상태 주문 목록을 먼저 보여준다.
- 시료관리/주문/모니터링 서브메뉴에 "0. 취소"를 추가하고, "0" 선택 시 "취소되었습니다" 안내 후 메인 메뉴로 돌아간다.

## 설계

- `controller/main_controller.py`의 `_handle_order_menu`에서 action `"2"`(승인)/`"3"`(거절)/`"4"`(취소) 분기 시작 부분에 `self.order_controller.list_pending()` 결과를 `console_view.show_orders(...)`로 먼저 출력한 다음, 기존처럼 `prompt_order_id()`를 호출한다. `list_pending()`/`show_orders()`는 Phase32/33에서 이미 만든 것을 그대로 재사용한다 — 새 controller/view 메서드 추가 없음.
- `_handle_sample_menu`/`_handle_order_menu`/`_handle_monitoring_menu`의 서브메뉴 프롬프트 문구에 "0. 취소"를 추가하고, `action == "0"`이면 `console_view.show_message("취소되었습니다.")` 후 `return`한다 (기존 각 메서드의 else 분기는 그대로 "잘못된 선택입니다"로 유지, 0만 취소로 별도 처리).
