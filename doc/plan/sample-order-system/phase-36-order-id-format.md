# Phase 36: 주문ID 포맷 변경 (O1 → ORD-001)

## 배경

주문ID 접두어 "O"(알파벳 대문자)가 숫자 "0"과 시각적으로 거의 구분이 안 돼, 콘솔에서 "O1"을 "01"로 착각해 order_id 입력이 실패하는 실사용 혼동이 발생했다.

## 목표

주문ID 포맷을 `O1`, `O2` ... 에서 `ORD-001`, `ORD-002` ... (하이픈 구분 + 3자리 0패딩, 4자리 넘어가면 자동으로 늘어남)로 바꾼다.

## 설계

- `controller/order_controller.py`의 `submit()`: `order_id = f"O{len(self.order_repo.find_all()) + 1}"` → `order_id = f"ORD-{len(self.order_repo.find_all()) + 1:03d}"`로 변경. 로직(순번 계산 방식)은 그대로, 포맷 문자열만 변경.
- 다른 모델/리포지토리/뷰 코드는 order_id를 그냥 문자열로 다루므로 (파싱하거나 접두어를 가정하는 코드 없음) 추가 변경 불필요 — 실제로 그런지 확인은 한다.
- 기존 테스트에서 `"O1"`, `"O2"`처럼 하드코딩된 주문ID 기대값을 쓰는 곳을 전부 새 포맷(`"ORD-001"`, `"ORD-002"`)으로 갱신한다. 테스트가 `order.order_id`로 직접 만든 값을 재사용하는 경우(하드코딩 아님)는 그대로 둔다.

## 완료 조건

- [x] `OrderController.submit()`이 `ORD-001`, `ORD-002` ... 형식으로 순번을 매기는지 테스트
- [x] 기존에 `"O1"`/`"O2"` 등을 하드코딩하던 테스트 전부 새 포맷으로 갱신 후 전체 테스트 통과
- [x] `python main.py`로 주문 접수 → 전체조회에서 `ORD-001` 형식으로 보이는지, 그 ID로 승인이 정상 동작하는지 수동 확인
