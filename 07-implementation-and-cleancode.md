# 미션2: 기능 구현 및 CleanCode/Commit 이력 관리

## 목표
정의된 도메인/메뉴 요구사항을 전체 구현하고, CleanCode 원칙과 의미 단위 커밋 이력을 관리한다.

## 배경
Action 단계에서 Plan.md의 계획에 따라 실제 기능을 구현한다. 이때 harness 문서(CLAUDE.md 등) 관리, 클린코드 원칙 준수, 커밋 이력 관리가 필요하다.

## 요구사항 상세
- 05번 문서에 정의된 도메인/메뉴 요구사항 전체 구현:
  - 시료관리(등록/조회/검색)
  - 주문(접수/승인/거절)
  - 모니터링(주문량-상태별 카운트, 재고량-여유/부족/고갈)
  - 출고처리(CONFIRMED -> RELEASE)
  - 생산라인(생산중 시료 + 대기큐, FIFO)
- CLAUDE.md 등 harness 문서 관리(프로젝트 규칙/구조를 문서에 반영)
- CleanCode 원칙 준수:
  - 중복 코드 제거
  - 명확한 네이밍
  - 계층 분리 유지(예: MVC 등 역할 분리)
- 의미 단위 커밋 이력 관리(기능/작업 단위로 커밋 분리)
