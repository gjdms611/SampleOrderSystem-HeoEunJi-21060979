# 도메인 모델 및 PRD 기반 요구사항 정의

## 목표
반도체 시료 생산주문관리 시스템(가상회사 S-Semi)의 도메인 모델, 주문 상태머신, 역할별 권한, 메인 메뉴 5종, 생산라인 계산식을 정의하고 이를 PRD.md에 문서화한다.

## 배경
SampleOrderSystem은 Agentic Engineering의 Explore 단계 산출물로 PRD.md를 작성해야 한다. PRD 작성에 앞서 도메인 모델과 핵심 규칙을 명확히 정의해야 한다.

## 요구사항 상세

### Sample(시료) 속성표

| 속성 | 설명 |
|---|---|
| 시료ID | 시료를 식별하는 고유 ID |
| 이름 | 시료 이름 |
| 평균생산시간 | 시료 1단위 생산에 소요되는 평균 시간 |
| 수율 | 정상시료 수 / 총생산시료 수 (예: 0.9) |

### Order(주문) 상태머신

```
RESERVED(접수)
  -> 거절: REJECTED(모니터링 제외, 정상 흐름 아님)
  -> 승인: 재고확인
       -> 재고충분: CONFIRMED(출고대기) -> 출고처리 -> RELEASE(출고완료)
       -> 재고부족: 생산라인 자동등록 + PRODUCING(생산중)
            -> 생산완료: CONFIRMED(출고대기) -> 출고처리 -> RELEASE(출고완료)
```

- RESERVED: 주문 접수 상태
- REJECTED: 거절된 주문. 모니터링 대상에서 제외되며 정상 흐름으로 취급하지 않음
- CONFIRMED: 승인 및 재고 확보 완료, 출고 대기 상태
- PRODUCING: 재고 부족으로 생산라인에 등록되어 생산 중인 상태
- RELEASE: 출고 완료 상태

### 역할별 권한표

| 역할 | 권한 |
|---|---|
| 고객 | 이메일로 시료 요청(시스템 외부 채널, 시스템 내 직접 입력 권한 없음) |
| 주문담당자 | 주문서 작성, 주문 접수(RESERVED 생성) |
| 생산담당자 | 개발시료 등록, 주문 승인/거절 결정 |

### 메인 메뉴 5종 기능 정의
1. 시료관리: 시료 등록 / 조회 / 검색
2. 주문: 주문 접수 / 승인 / 거절
3. 모니터링: 주문량(상태별 카운트), 재고량(여유/부족/고갈) 확인
4. 출고처리: CONFIRMED 상태 주문을 RELEASE 상태로 전환
5. 생산라인: 현재 생산 중인 시료 및 대기큐(FIFO) 확인

### 생산라인 계산식
- 실생산량 = ceil(부족분 / 수율)
- 총생산시간 = 평균생산시간 * 실생산량
